# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| latest  | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do NOT open a public GitHub issue.**

Instead, please email: **security@carlgra.com**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgement**: within 48 hours
- **Initial assessment**: within 5 business days
- **Fix or mitigation**: targeting 30 days for critical issues

## Disclosure Policy

We follow coordinated disclosure. We will:
1. Confirm the vulnerability and determine affected versions
2. Develop and test a fix
3. Release a patched version
4. Publish a security advisory on GitHub

We appreciate your help keeping kafka-lag-exporter and its users safe.
